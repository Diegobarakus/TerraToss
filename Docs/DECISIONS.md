# TerraToss — Decisions

## D-001 — Unity as the client engine

Status: Approved.

Unity 6.3 LTS with URP is used for Android and iOS.

## D-002 — Claude Code is the only development AI

Status: Approved.

Unity AI is not used.

UnityMCP is a technical bridge, not a second AI.

## D-003 — Placeholder-first development

Status: Approved.

Use Unity primitives and basic URP materials until the core mechanic is validated.

## D-004 — Android-first development

Status: Approved.

Development and device testing begin on Android while preserving iOS compatibility.

## D-005 — Fictional projectile physics

Status: Approved.

Real geographic coordinates and distances are used, but projectile range and flight are deliberately fictional and game-oriented.

## D-006 — Family-friendly presentation

Status: Approved.

No realistic military presentation, killing, or human targets.

## D-007 — Camp instead of continuous tracking

Status: Approved.

Future multiplayer targets a stored approximate camp location, not continuous background tracking.

## D-008 — Server-authoritative multiplayer later

Status: Planned.

When multiplayer is implemented, the server calculates final shot results and stores deterministic replay data.
